Objects in javascript

Here you wil learn the 2 types of objects, class objects, and class literals

Reusability:


Classes/Constructors: Ideal for creating multiple instances with similar properties and methods. Promotes code reuse.

Object Literals: Best for creating individual objects with unique properties. Less reusable

When to Use Each Approach

Classes/Constructors: Use when you need to create multiple instances with similar properties and methods, and especially when you need inheritance.
Object Literals: Use for simpler structures, configuration objects, or when creating a single-use object with a fixed set of properties and methods.

Encapsulation, Absatraction, Inheritance, Polymorphism

Class Objects

This approach involves using constructors, which are special functions used to create and initialize objects.

class person {
constructor(name, age) {
this.name = name; //the parameter name will now be inside of this.name
this.age = age;
}
greet(){
console.log(`Hello my name is ${this.name} and i am ${this.age} old`)
}
}

class student extends person{
study(){

console.log(`${this.name} is studying`)
}
}
// Here i am setting a person with person values
const rene = new person("Rene", 21)
const michelle = new person("Michelle",19)

rene.greet(); //Hello my name is Rene and i am 21 old
michelle.greet() //Hello my name is Michelle and i am 19 old
//This might not be the best example but i am showing how can a class extend
const estudiante = new student("Michelle")
estudiante.study() //Michelle is studying

Objects are passed by reference. This means when we pass a variable assigned to an object into a function as an argument, the computer interprets the parameter name as pointing to the space in memory holding that object. As a result, functions which change object properties actually mutate the object permanently (even when the object is assigned to a const variable).

This means when we can change the value of a object using their parameter, or (mutate) but we cannot add new values.
