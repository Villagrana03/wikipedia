Likes Vs Dislikes

YouTube had a like and a dislike button, which allowed users to express their opinions about particular content. It was set up in such a way that you cannot like and dislike a video at the same time. There are two other interesting rules to be noted about the interface: Pressing a button, which is already active, will undo your press. If you press the like button after pressing the dislike button, the like button overwrites the previous "Dislike" state. The same is true for the other way round.

function likeOrDislike(buttons) {

//This variable keeps track of the current state, which can be "Nothing", "Like", or "Dislike".


let currentState = "Nothing"

// On each iteration, the variable button will hold the current element from the buttons array.


for(button of buttons){

if (button === currentState) checks if the current button is equal to currentState. If they are equal, it means the button is being pressed again to undo the current state. currentState = "Nothing"; sets the currentState to "Nothing", indicating that the previous state has been undone


if(button === currentState){
currentState = "Nothing"
} else{
currentState = button
}
}
return currentState
}

Is it a letter?

Write a function, isItLetter or is_it_letter or IsItLetter, which tells us if a given character is a letter or not.

function isItLetter(character) {

// ^[A-Za-z]$: This is a regular expression pattern.

^ asserts the start of the string.
[A-Za-z] matches any uppercase or lowercase letter.
$ asserts the end of the string.
Together, this pattern ensures that the string consists of only one letter, whether uppercase or lowercase.

// It takes a string argument (character) and checks if that string matches the regular expression pattern.
If the string matches the pattern, .test() returns true; otherwise, it returns false.

return /^[A-Za-z]$/.test(character);
}
